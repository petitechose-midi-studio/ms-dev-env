# Windows Native Toolchain (PlatformIO `native`) via Zig

Date: 2026-02-14

Context:
- PlatformIO `platform = native` does not ship a toolchain; it relies on `gcc/g++` being on `PATH`.
- On Windows, relying on a system MinGW can be fragile (PATH/DLL conflicts). We already bundle Zig and Ninja.

Goal:
- After `uv run ms setup --yes`, Windows users can run PlatformIO native unit tests reliably.
- Linux/macOS remain system-toolchain based.

Decisions:
- Windows only: provide `gcc/g++/ar/ranlib` via Zig-backed wrappers in `tools/bin`.
- Windows only: set a safe default `SCONSFLAGS=-j1` for PlatformIO (prevents Zig/LLVM OOM in parallel builds).
- Windows only: limit Ninja parallelism in `ms build <app> --target native` (default `-j 1`, override via `MS_WINDOWS_NATIVE_JOBS`).

Implementation:
- `ms` generates wrappers:
  - cmd wrappers for cmd/PowerShell (`gcc.cmd`, `g++.cmd`, ...)
  - bash wrappers without extension for Git Bash (`gcc`, `g++`, ...)
- Activation scripts (`tools/activate.*`) export `SCONSFLAGS=-j1` on Windows.
- `ms check` validates the bundled C/C++ toolchain by compiling a trivial C++ TU through the wrapper.
